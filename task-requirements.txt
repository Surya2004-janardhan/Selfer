# SELFER: Local-First Autonomous Agent Framework

## Overview
`selfer` is a professional-grade, local-first autonomous agent framework. It empowers developers with a multi-agent system that operates entirely within their project directory, ensuring data privacy and context-aware intelligence.

## Core Features
- **Local-First Architecture**: Prioritizes privacy and speed by running LLMs locally.
- **Autonomous Multi-Agent System**: Utilizes 15+ specialized agents for complex task orchestration.
- **Plugin & Skill System**: Extensible via Markdown-based skill files.
- **Bi-Interface Support**: Advanced CLI and Telegram Bot integration.

## CLI Commands
- `selfer init`: Initializes the `.selfer` workspace, scans the directory, and gains initial project knowledge.
- `selfer start`: Launches the interactive agent session (CLI or CLI + Telegram).
- Custom Commands: Support for `/<command>` within the chat interface and `--tag` for operational targeting.

## Agent Architecture
Selfer orchestrates workflows through a Master/Router architecture, utilizing:
1.  **Plan-Agent**: Decomposes tasks into actionable steps.
2.  **CLI-Agent**: Manages terminal interaction.
3.  **Git-Agent**: Handles semantic versioning and repository management.
4.  **File-Agent**: Manages file system operations.
5.  **Web-Agent**: Performs standard web searches.
6.  **Code-Agent**: Analyzes and understands logic/architecture.
7.  **Review-Agent**: Performs code quality reviews.
8.  **Edits-Agent**: Executes code modifications.
9.  **Routing-Agent**: Decisions on agent delegation and flow.
10. **Permission-Agent**: Manages security and user approvals.
11. **Telegram-Agent**: Bridges communication with the Telegram API.
12. **Context-Agent**: Provides deep context to other agents.
13. **Sub-Process-Agent**: Manages background task execution.
14. **Process-Tracking-Agent**: Monitors task progress and status.
15. **Error-Recovery-Agent**: Automatically handles and recovers from failures.
16. **Error-Tracker-Agent**: Logs and analyzes repetitive errors.
17. **Browser-Agent**: Deep web navigation and proxy-based research.
18. **Memory-Agent**: Manages session memory and snapshots.

## Intelligence & Efficiency
- **Token-First Approach**: Optimizes context usage to minimize costs and maximize model performance.
- **Batch Processing**: Automatically divides large tasks into efficient sub-processes.
- **Resonant Recovery**: Features advanced self-correction logic; if an error persists beyond 3 attempts, it proactively seeks user context.
- **Checkpointing**: Saves project states as snapshots in `.selfer` for reliable rollback and context retention.

## UI/UX Standards
- **Premium Aesthetics**: High-end CLI design (Green/Blue themes) with micro-animations.
- **Reasoning Logs**: Transparent display of agent planning and execution steps.
- **Thread Safety**: Queues user queries during active tasks without disrupting the ongoing plan.

## Technical Goals
The final product must be an industry-standard framework comparable to state-of-the-art tools like Cursor, Claude Code, and Antigravity, delivering superior accuracy and reliability.
